    Create a new file at src/app/api/sightings/route.ts. This is a Next.js App Router route handler.

    Imports: NextRequest, NextResponse from 'next/server'; auth from '@/auth'; db from '@/lib/db'; sightings, collections from '@/lib/db/schema'; eq, and from 'drizzle-orm'.

    Export an async POST function accepting NextRequest.

    Auth check: call auth(). If session?.user?.id is falsy, return 401 JSON { error: 'Unauthorized' }.
    Extract userId as session.user.id. If TypeScript requires a cast to access id, cast session.user as { id: string }.

    Parse body: wrap req.json() in try/catch. On parse error return 400 JSON { error: 'Invalid JSON' }.
    Destructure { speciesId, gridSquare } from body.

    Validate speciesId: must be a non-empty string. Return 400 { error: 'speciesId is required' } if not.
    Validate gridSquare: must match /^[A-Z]{2}[0-9]{2}$/ (2 uppercase letters + 2 digits). Return 400 { error: 'Invalid grid square' } if not. This is the ONLY location data accepted — per D-02 and D-03.

    Insert sighting row: db.insert(sightings).values({ userId, speciesId, gridSquare }). Await this.

    Upsert collection:
    - Query existing: db.select({ sightingCount: collections.sightingCount }).from(collections).where(and(eq(collections.userId, userId), eq(collections.speciesId, speciesId))).limit(1). Await.
    - If existing.length === 0: insert { userId, speciesId, sightingCount: 1 }. Set sightingCount = 1, firstSighting = true.
    - Else: sightingCount = existing[0].sightingCount + 1. Update collections: set({ sightingCount, lastSightedAt: new Date() }) where userId and speciesId match. Set firstSighting = false.

    Return 200 JSON { sightingCount, firstSighting }.

    Do NOT use onConflictDoUpdate for the upsert — use the explicit select+branch pattern above for clarity (per research §2).
  </action>
