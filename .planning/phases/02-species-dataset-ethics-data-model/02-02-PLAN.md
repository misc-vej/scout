Create "data/seed.ts" as an idempotent upsert script (per RESEARCH.md §7 pattern).

The script must:
1. Import "db" from "../src/lib/db"
2. Import "species" from "../src/lib/db/schema"
3. Import "seedData" from "./species-seed"
4. Define an async "seed()" function that loops over seedData and for each entry calls: db.insert(species).values(entry).onConflictDoUpdate({ target: species.scientificName, set: { commonName, rarityTier, sensitivityLevel, canBeShiny, seasonLockStart, seasonLockEnd, taxonomyGroup, description, tvk, updatedAt: new Date() } })
5. Log progress: "Seeding N species..." before the loop, "Seed complete." after
6. Call process.exit(0) on success, and catch errors with console.error + process.exit(1)
7. Call seed() at the bottom of the file

The upsert conflict target is "species.scientificName" because that column has the ".unique()" constraint in the schema. This makes re-running the seed script safe — it updates existing rows rather than inserting duplicates.

After writing the file, run the seed script: "npm run db:seed"

It must complete without errors and print "Seed complete." then exit 0.
