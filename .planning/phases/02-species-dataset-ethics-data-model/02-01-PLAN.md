  <name>Task 2: Generate and apply Drizzle migration to Neon</name>
  <files>drizzle/</files>
  <action>
Run drizzle-kit generate and drizzle-kit migrate to create and apply the migration.

The migration commands must use DATABASE_URL_UNPOOLED (the direct connection string) for DDL operations — the pooled DATABASE_URL does not support schema migrations reliably on Neon.

Read DATABASE_URL_UNPOOLED from .env.local, set it as DATABASE_URL for the duration of the migration commands, then run:

```
npx drizzle-kit generate
DATABASE_URL=$(grep DATABASE_URL_UNPOOLED .env.local | grep -o '"[^"]*"' | tr -d '"') npx drizzle-kit migrate
```

If DATABASE_URL_UNPOOLED is set without quotes in .env.local (as a bare value rather than quoted string), adapt the extraction accordingly. The drizzle.config.ts reads `process.env.DATABASE_URL`, so the environment variable must be set for the migrate command.

Do NOT use `drizzle-kit push` — it has TTY interaction issues in non-interactive shells (per CONTEXT.md established patterns).

After migrate completes, verify the species table exists by querying Neon:
