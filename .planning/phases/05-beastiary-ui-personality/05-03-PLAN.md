  <action>
    Replace the Phase 4 beastiary page entirely. The new page is a server component with this structure:

    Imports:
    - `redirect` from `next/navigation`
    - `auth` from `@/auth`
    - `db` from `@/lib/db`
    - `species`, `collections` from `@/lib/db/schema`
    - `eq`, `asc` from `drizzle-orm`
    - `BeastiaryCard` from `@/components/beastiary/BeastiaryCard`

    Data fetching:
    1. Auth guard: `const session = await auth()`. If no session/user, `redirect('/auth')`.
    2. `const userId = (session.user as { id: string }).id`
    3. Fetch all species ordered by taxonomyGroup asc, commonName asc (same as Phase 4)
    4. Fetch user collections — select `speciesId`, `sightingCount`, AND `personalityTrait` (new field added by 05-02):
       ```
       db.select({ speciesId: collections.speciesId, sightingCount: collections.sightingCount, personalityTrait: collections.personalityTrait })
         .from(collections)
         .where(eq(collections.userId, userId))
       ```
    5. Build `collectedMap: Map<string, { sightingCount: number; personalityTrait: string | null }>` from the results

    Progress bar (retained from Phase 4, D-10):
    - `totalCollected = collectedMap.size`, `totalSpecies = allSpecies.length`
    - Same progress bar markup as Phase 4: `h-1.5 rounded-full bg-green-500` width calculated as percentage

    Taxonomy filter tabs (D-09):
    - Display group tabs for: Birds / Mammals / Reptiles & Amphibians / All
    - For MVP, implement as static visual tab row (no interactivity required — client-side filtering can be a future phase). Render all species grouped under section headings as per current Phase 4 behaviour, with a tab row above for visual chrome.
    - Tab row: `flex gap-2 mb-4 overflow-x-auto`. Each tab: `px-3 py-1 rounded-full text-sm`. Show "All" as the active/highlighted tab (green background) since filtering is static.

    Card grid (D-09):
    - Remove the section-based list layout from Phase 4
    - Render a single flat grid (or per-group sections each with a grid — developer choice based on what looks cleanest):
      - Recommended: Keep the group sections as `<section>` headings, but replace the `flex flex-col gap-2` list inside each with a card grid
      - Grid class: `grid grid-cols-2 md:grid-cols-3 gap-3`
    - For each species in the group, render:
