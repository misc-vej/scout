    The route must implement exactly the handler shown in RESEARCH.md §3:

    - Import `NextRequest`, `NextResponse` from `next/server`
    - Import `auth` from `@/auth`
    - Import `db` from `@/lib/db`
    - Import `collections` from `@/lib/db/schema`
    - Import `eq`, `and` from `drizzle-orm`
    - Define `ALLOWED_TRAITS` as a const array: `['Brave', 'Sneaky', 'Chill', 'Grumpy', 'Curious', 'Dramatic', 'Wise', 'Chaotic']`
    - Export `async function PATCH(req: NextRequest, { params }: { params: Promise<{ speciesId: string }> })`

    Handler logic:
    1. `const session = await auth()` — if no session or no user ID, return 401
    2. `const userId = (session.user as { id: string }).id`
    3. `const { speciesId } = await params` — must await (Next.js 16)
    4. Parse request body with try/catch — return 400 on invalid JSON
    5. Validate `body.trait` against ALLOWED_TRAITS — return 400 with list of valid traits if invalid
    6. Run `db.update(collections).set({ personalityTrait: body.trait }).where(and(eq(collections.userId, userId), eq(collections.speciesId, speciesId))).returning({ personalityTrait: collections.personalityTrait })`
    7. If result array is empty, return 404 `{ error: 'Species not in collection' }`
    8. Return 200 `{ personalityTrait: result[0].personalityTrait }`

    See RESEARCH.md §3 for the full typed implementation to use as reference.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client → PATCH /api/collections/[speciesId]/personality | Unauthenticated or forged requests; speciesId path param from client |
| API → Neon | SQL update using userId + speciesId — must use AND condition to prevent cross-user updates |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-02-01 | Spoofing | PATCH /personality — auth check | mitigate | auth() called first; 401 returned if no session.user.id |
| T-05-02-02 | Tampering | trait value from request body | mitigate | ALLOWED_TRAITS allowlist validated before any DB write; 400 on invalid |
| T-05-02-03 | Tampering | speciesId param — IDOR risk | mitigate | WHERE clause includes both userId AND speciesId; can only update own collection rows |
| T-05-02-04 | Information Disclosure | 404 on species not in collection | accept | Reveals that speciesId doesn't belong to this user, but speciesId is a non-secret UUID |
</threat_model>
