<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser WebAuthn -> Supabase | Public-key credential created/asserted client-side; Supabase stores/verifies |
| client -> profiles update | User updates own passkey_prompted_at (RLS-guarded) |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-01 | Spoofing | passkey registration | high | mitigate | Use Supabase native `signInWithPasskey` (verifies WebAuthn assertion server-side); never roll custom WebAuthn |
| T-02-02 | Tampering | passkey_prompted_at update | medium | mitigate | Update goes through RLS UPDATE policy (`auth.uid() = id`); a user can only set their own flag |
| T-02-03 | Denial of Service | unsupported-browser crash | medium | mitigate | Detect `window.PublicKeyCredential` before showing passkey UI; hide option entirely if absent (no thrown error) |
| T-02-04 | Repudiation | email verification | low | accept | D-07 soft prompt by design; verification is surfaced but not gated. Banner reflects `email_confirmed_at` truthfully |
</threat_model>
