Implement the passkey setup prompt (D-03) — shown immediately after email/password signup — and the passkey section on the profile page (D-04). | T-02-02 | Information Disclosure | profile page — session.user.email rendered | accept | Email is the user's own data; no other user's data is exposed; query is scoped to session.user.id |
| T-02-SC | Tampering | npm packages (no new installs this plan) | accept | No new npm installs in this plan — schema and auth packages from Plan 01 are already in package-lock.json |
